# Ms. Y — Portfolio (GitHub Pages)

A single-file portfolio site. No build step, no dependencies — just `index.html`.

## Deploy to GitHub Pages

1. Create a repo named `YOUR-USERNAME.github.io` (site lives at the root URL) — or any repo name (site lives at `/repo-name/`).
2. Add `index.html` and push to the `main` branch.
3. Repo → **Settings → Pages** → Source: **Deploy from a branch** → Branch: `main` / `/ (root)` → Save.
4. Wait ~1 minute. Site is live at `https://YOUR-USERNAME.github.io/`.

## Before publishing — replace the placeholders

- `YOUR-USERNAME` (GitHub links, 2 places)
- `YOUR-PROFILE` (LinkedIn)
- `you@example.com`
- Case study links (`href="#"`) → point to blog posts or repo READMEs
- Stats/certs in the proof strip → keep them honest

## Custom domain (optional, ~$10/yr)

Settings → Pages → Custom domain → add `msy.dev` (or similar), then set the
DNS records GitHub shows you. HTTPS is automatic.
